# 전략 설계 메모

## 핵심 아이디어

- KOSPI200을 기본 universe로 사용한다.
- 가격, 시가총액, 거래대금, 차트 우상향 조건으로 후보군을 줄인다.
- 후보군이 보통 50개 전후라면 초당 5회 REST 조회 제한에서도 10초 안팎으로 한 cycle을 돌 수 있다.
- 각 종목은 20호가 스냅샷을 1회만 조회한다.
- 호가의 매수잔량/매도잔량을 1:1로 상쇄하여 예상가를 계산한다.
- 예상수익률 상위 10% 중 절대 기준을 통과한 종목만 매수한다.
- 매수 체결 즉시 예상가 - 1틱에 지정가 매도를 넣는다.
- 13:00까지 미체결 또는 보유수량이 남아 있으면 시장가 강제청산한다.

## 상태 흐름

```text
NO_POSITION
→ SCANNING
→ BUYING
→ SELLING
→ 13:00 FORCE_SELLING
→ STOPPED
```

## 실전 전 필수 확인

1. 키움 REST API 20호가 endpoint와 응답 필드 매핑
2. 주문 endpoint와 주문 구분값
3. 매수 체결 확인 방식
4. 미체결 주문 취소 방식
5. 시장가 매도 방식
6. KOSPI200 구성종목 데이터 출처
7. KRX 최신 호가단위
8. 수수료/세금/슬리피지 반영 방식
