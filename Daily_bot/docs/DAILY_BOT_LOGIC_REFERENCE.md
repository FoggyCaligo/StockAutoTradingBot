# Daily Bot Logic Reference

이 문서는 현재 실코드와 백테스트가 어떤 개념으로 맞물려 돌아가는지 설명하는 기준 문서다. 세부 진실원천은 [main.py](/C:/Users/bigla/OneDrive/Documents/GIT/StockAutoTradingBot/Daily_bot/main.py), [settings.yaml](/C:/Users/bigla/OneDrive/Documents/GIT/StockAutoTradingBot/Daily_bot/config/settings.yaml), [replay_market_traces.py](/C:/Users/bigla/OneDrive/Documents/GIT/StockAutoTradingBot/Daily_bot/backtest/replay_market_traces.py)다.

## 0. 처음 보는 사람을 위한 읽는 순서

이 프로젝트를 처음 보는 사람이라면 아래 순서로 보면 된다.

1. 유니버스가 어떻게 정해지는가
2. 60초 스캔 배치가 어떻게 돌아가는가
3. 호가에서 기대수익률을 어떻게 계산하는가
4. 후보 중 실제로 무엇을 사는가
5. 목표가, 마감청산, 일손실 제한은 어떻게 작동하는가
6. 백테스트는 이 과정을 어떻게 복원하는가

이 전략은 가격 시계열만 보는 전략이 아니라, 호가 스냅샷과 계좌 상태를 함께 쓰는 전략이다.

## 1. 전략 목표

데일리 봇은 당일 장중에 짧은 시간 안에 체결될 가능성이 있는 미세한 기대수익 기회를 찾는 전략이다.

핵심 목표
- KOSPI 전체 중 유동성이 충분한 종목만 본다.
- 60초마다 호가를 다시 읽고 기대수익률을 다시 계산한다.
- 기대수익률이 0.7% 이상인 후보만 자본과 슬롯 구조 안에서 진입시킨다.
- 매수 직후 목표가 매도를 걸고, 남은 포지션은 15:00에 강제정리한다.

## 2. 유니버스 구성

현재 유니버스는 고정 CSV가 아니라 당일 조회한 KOSPI 전체 종목에서 바로 만들어진다.

- 시장: `KOSPI`
- 최소 시가총액: `2500억 원`
- 최소 거래대금: `30억 원`
- 추세 필터: 코드 경로는 있지만 현재 운영값은 `OFF`

이 뜻은 단순하다.
- 전략은 시장 전체를 넓게 보되
- 거래하기 어려운 종목은 처음부터 제거한다.

## 3. 시간대 운영 흐름

- `08:55`: 사전 준비 시작
- `09:10`: 전일 잔존 포지션이나 미체결 주문이 있으면 먼저 정리
- `09:30`: 신규 매수 스캔 시작
- `11:30`: 신규 매수 종료
- `15:00`: 남은 포지션 강제청산
- `15:15`: 체결/잔고 재정합
- `15:20`: 당일 루프 종료

09:10 정리 단계는 전일 잔존 리스크를 끌고 신규 매매를 시작하지 않기 위한 운영 안전장치다.

## 4. 스캔과 배치 개념

전략의 기본 단위는 “개별 종목”이 아니라 “한 번의 전체 스캔 배치”다.

- 스캔 주기: `60초`
- 각 스캔에서 유니버스 전체 종목을 다시 순회
- 각 종목의 호가 스냅샷에서 기대수익률 계산
- 해당 배치 전체 후보를 놓고 실제 매수 조합 선택

따라서 전략을 이해하거나 재구성할 때는 “이 종목이 좋았나”보다 “그 시각에 무엇이 같이 후보였고, 계좌는 무엇을 살 수 있었나”를 함께 봐야 한다.

## 5. 호가 모델과 기대수익률 계산

현재 기대수익률 계산은 차트 지표가 아니라 호가 상쇄 모델이다.

1. 매수호가와 매도호가를 읽는다.
2. 양쪽 잔량을 1:1로 차감하면서 균형이 형성되는 지점을 찾는다.
3. 그 지점의 매수/매도 프론티어 중간값을 `expect_price`로 본다.
4. 여기서 `sell_tick_offset = 1`틱을 뺀 목표 매도가를 만든다.
5. 목표 매도가와 현재가 차이로 `expect_revenue_percent`를 계산한다.

개념식

```text
expect_revenue_percent = (target_sell_price - current_price) / current_price * 100
```

이 계산이 전략의 중심인 이유
- 현재 전략은 이동평균, RSI 같은 누적형 차트 지표보다
- “지금 호가 구조가 어디로 균형을 잡으려 하는가”를 더 중요하게 본다.

## 6. 현재 호가 감쇠 모델

최근 변경으로 실코드와 백테스트 모두, 먼 호가 잔량을 원본 그대로 1:1로 믿지 않는 구조를 쓴다.

- 매수/매도 양쪽 모두 같은 방식으로 감쇠
- 1단계 호가 가중치: `1.0`
- 마지막 단계 호가 가중치: `0.1`
- 중간 단계는 선형으로 감소

즉 가까운 호가잔량은 거의 그대로 반영하고, 현재가에서 먼 잔량일수록 영향력을 작게 본다.

재구성 시 주의점
- 감쇠는 후처리 필터가 아니다.
- 감쇠된 호가로 `expect_price` 자체를 다시 계산한다.
- 감쇠를 빼면 기대수익률, 후보 수, 실제 매수 결과까지 달라진다.

## 7. 현재 진입 기준

현재 운영에서 실제로 켜져 있는 진입 기준은 아래다.

- `top_ratio = 0.25`
- 기대수익률 기준 `0.7%`
- fallback `OFF`
- 전일 상승률 상한 `1.0%`
- 스프레드 상한 `OFF`
- 직전 스캔 급등 억제 `OFF`
- 매도호가 잔량비율 필터 `OFF`
- 추세 필터 `OFF`

즉 현재 전략은 “강한 기대수익 후보만 받고, 그중 상위 25%만 남기고, 전일 과열 종목만 추가로 거른다”는 구조다.

## 8. 실제 종목 선택 방식

필터를 통과한 종목을 기대수익률 순서대로 무조건 사는 구조는 아니다.

실코드와 백테스트 모두 아래 요소를 함께 본다.

- 현재 남은 현금
- 슬롯 수와 슬롯별 예산
- 종목별 실제 주문 가능 수량
- 스캔당 신규 진입 상한 `3`
- 총 보유 하드 상한 `10`

따라서 “순위 계산기”와 “실제 매수 조합 선택기”는 분리된 단계다.

## 9. 자본과 슬롯 구조

현재 전략은 고정 3종목 전략이 아니다.

- 최소 슬롯 수: `3`
- 최대 슬롯 수: `10`
- 슬롯 계산 단위: `500만 원`
- 종목당 목표 예산 비율: `50%`
- 총 보유 하드 상한: `10`

여기에 별도로 `max_buy_count = 3`이 걸려서, 한 스캔에서 너무 많은 신규 진입이 한 번에 발생하는 것을 막는다.

즉
- 자본이 커지면 총 보유 수는 늘 수 있고
- 한 스캔에서 새로 진입하는 속도는 최대 3개로 묶인다.

## 10. 재매수 구조

현재 `allow_refill_empty_slots = true`이므로, 중간에 포지션이 정리되어 빈 슬롯이 생기면 다음 스캔에서 다시 신규 진입이 가능하다.

즉 현재 구조는
- 한 세트를 다 팔 때까지 멈추는 구조가 아니라
- 비는 자리를 다시 채우는 구조다.

## 11. 청산 구조

현재 청산 우선순위는 아래다.

1. 매수 직후 목표 지정가 매도
2. 장중에는 손절 없음
3. 장 마감 `15:00` 강제청산
4. 마감 이후 `15:15` 체결 재정합

현재 장중 손절이 완전히 꺼져 있다는 점이 중요하다.

- `risk.stop_loss_percent = 0.0`
- `risk.stop_loss_tick_count = 0`
- `risk.stop_loss_tick_multiplier = 0.0`

즉 현재 전략은 “장중 손실을 빠르게 잘라낸다”보다 “목표가 체결 또는 마감 정리”에 더 무게를 둔다.

## 12. 계좌 단위 방어

종목 손절은 꺼져 있지만 계좌 단위 방어는 남아 있다.

- `daily_loss_limit_percent = 10.0`

즉 전략은 개별 종목 손절 대신, 당일 전체 계좌 손실이 과도해질 때 신규 진입을 막는 구조를 유지한다.

## 13. 기록 구조

현재 전략은 나중에 복원 가능한 로그를 남기는 것을 매우 중요하게 본다.

주요 기록
- `market_traces`: 각 스캔 배치의 후보 계산 결과와 raw 호가
- `signals`: 실제 선택 여부
- `orders`: 주문 이력
- `fills`: 체결 이력
- `account_traces`: 계좌 상태

이 구조 덕분에 아래 질문들에 답할 수 있다.

- 그 시각에 어떤 후보들이 동시에 있었는가
- 그중 무엇이 실제로 선택되었는가
- 계좌 상태 때문에 무엇이 탈락했는가
- 이후 목표가 체결로 끝났는가, 마감청산으로 끝났는가

## 14. 백테스트 재구성 개념

현재 백테스트는 단순 수익률 계산기가 아니라 “실거래 판단 과정 재구성기”에 가깝다.

핵심 재구성 원칙
- `market_traces.raw_json`에서 당시 호가를 다시 읽는다.
- `scan_cycle_at` 기준으로 같은 스캔 배치를 묶는다.
- 신규 진입 후보는 `scan_candidate` 단계 기준으로 본다.
- 기대수익률 계산은 실코드와 같은 감쇠 호가 모델을 쓴다.
- 실제 매수는 같은 자본/슬롯/조합 선택 로직을 사용한다.
- 목표가 도달 시 백테스트 청산가는 `target_price`로 고정한다.

재구성 체크리스트
- 당시 유니버스는 무엇이었는가
- 같은 배치의 후보는 무엇이었는가
- 감쇠된 호가로 계산한 기대수익률은 얼마였는가
- 그중 무엇이 필터를 통과했는가
- 당시 몇 슬롯이 비어 있었는가
- 어떤 조합을 실제로 살 수 있었는가
- 목표가 체결 또는 마감청산 중 무엇으로 끝났는가

## 15. 백테스트와 실거래가 아직 다를 수 있는 지점

아래는 현재도 완전 일치시키기 어려운 부분이다.

- 60초 스캔 사이에 잠깐 찍고 내려온 장중 고가/저가
- 부분체결과 분할청산
- 취소 후 재주문 세부 순서
- 브로커 내부 체결 우선순위
- 로그 누락 또는 API 지연

그래서 현재 백테스트는 “실거래의 완전 복사본”이라기보다, “현재 전략 구조를 유지한 시뮬레이션 엔진”으로 보는 편이 맞다.

## 16. 현재 기본 설정 요약

- 시장: KOSPI 전체
- 유니버스 필터: 시총 `2500억` 이상, 거래대금 `30억` 이상
- 스캔: `60초`
- 기대수익률 기준: `0.7`
- 랭킹 컷: `상위 25%`
- fallback: `OFF`
- 호가 감쇠: 매수/매도 모두 `1.0 -> 0.1`
- 스캔당 신규 진입 상한: `3`
- 총 보유 상한: `10`
- 재매수: `허용`
- 전일 상승 상한: `1.0%`
- 장중 손절: `OFF`
- 일손실 제한: `10%`
