# Daily Bot Strategy Design

## 1. 전략 목표

Daily Bot의 목표는 KOSPI200 안에서 장중 짧은 호가 왜곡과 수급 불균형을 빠르게 포착해 짧게 진입하고, 같은 날 안에 정리하는 것이다.

핵심 전제는 다음과 같다.

- 20호가 정보만으로도 단기 예상가를 추정할 수 있다.
- 후보를 너무 좁게 자르기보다 넓게 본 뒤, 뒤쪽 필터와 자본 배분으로 진입 품질을 관리하는 편이 더 낫다.
- 수익 극대화보다 손실 확산 방지와 복기 가능성이 더 중요하다.

## 2. 후보군 생성

현재 기본 유니버스 조건은 다음과 같다.

- 대상: `KOSPI200`
- 최소 시가총액: `2500억`
- 최소 거래대금: `30억`
- 추세 필터: 비활성화

최근 방향은 "유니버스 필터를 너무 빡빡하게 두지 않는다" 쪽이다.  
예전보다 후보군은 넓게 보고, 뒤쪽 필터와 슬롯 구조로 실제 진입만 조절한다.

## 3. 예상가 계산

예상가는 20호가 기반으로 계산한다.

1. 각 종목의 20호가 스냅샷을 조회한다.
2. 매수/매도 잔량 균형이 맞는 가격대를 찾는다.
3. 그 균형대의 중간값을 예상가로 잡는다.
4. 호가 단위에 맞춰 반올림한다.

목표 매도가는 아래처럼 계산한다.

```text
target_sell_price = expect_price - 1 tick
```

기대수익률은 다음 기준으로 계산한다.

```text
(target_sell_price - current_price) / current_price * 100
```

## 4. 현재 매수 필터

현재 실운용 기준 주요 필터는 아래와 같다.

- `top_ratio = 0.20`
- `max_spread_percent = 0.7`
- `min_expected_return_percent = 0.30`
- `spread_expected_return_multiplier = 1.2`
- `max_prev_day_change_percent = 7.0`
- `max_intraday_jump_from_prev_scan_percent = 1.0`

실제 기대수익률 하한은 고정 `0.30%`가 아니라 아래 식으로 결정된다.

```text
required_expected_return = max(0.3, spread_percent * 1.2)
```

즉 스프레드가 넓은 종목일수록 더 높은 기대수익률을 요구한다.

추가로 현재는 아래 조건도 함께 본다.

- 목표 매도가가 현재가보다 높아야 함
- 이미 보유 중인 종목과 겹치지 않아야 함
- 거래가능현금과 슬롯 계획 안에서 1주 이상 매수 가능해야 함

## 5. 당일 급등 추격 방지

현재는 최소한의 당일 급등 추격 방지를 켜둔 상태다.

- 기준: 직전 스캔 대비 `+1.0%` 이상 상승한 후보 제외

이 필터는 최근 리플레이에서 제외 건수 자체는 많지 않았지만, 실거래 기준으로는 급격히 튄 종목의 뒤늦은 추격을 줄이는 안전장치 역할을 한다.

## 6. 자본 배분

현재 슬롯 계획은 아래 기준으로 움직인다.

- 최소 슬롯 수: `3`
- 최대 슬롯 수: `10`
- 슬롯 기준 금액: `500만원`
- 종목당 최대 예산: `500만원`

세션 시작 시점의 거래가능현금을 기준으로 슬롯 수를 계산하고, 종목당 예산을 정한 뒤 그 범위 안에서만 진입한다.

방향성은 "후보는 여러 개 보되, 특정 종목에 과하게 몰지 않는다"에 가깝다.

## 7. 주문금액 대비 호가잔량 필터

현재 실거래에는 주문금액 대비 호가잔량 필터가 포함되어 있다.

- 기준: `planned order amount <= top-5 ask depth amount * 0.30`

의도는 다음과 같다.

- 내 주문이 얇은 매도호가를 밀어 올리는 상황 방지
- 체결 직후 평균단가가 나빠지는 상황 완화
- 나중에 빠져나올 때 유동성이 너무 얇은 종목 회피

중요:

- 이 필터는 현재 코드에는 반영되어 있다.
- 하지만 과거 `market_traces`에는 top-5 매도잔량 금액이 저장되지 않았기 때문에, 옛 구간 백테스트에서는 완전 재현이 어렵다.

## 8. 손절과 청산

현재 활성 손절 기준은 다음과 같다.

- 손절선: `매수가 대비 -1.0%`

운용 흐름은 아래와 같다.

1. 매수 체결 확인
2. 목표가 매도 주문 등록
3. 손절 조건 충족 시 즉시 청산
4. `15:00` 이후 남은 포지션은 시장가 기준 강제 청산
5. `15:15` 이후 브로커 체결 내역으로 기록 보정

강제 청산을 시장가로 둔 이유는, 당일 안에 무조건 비우는 구조라면 마감 청산에서 지정가 미체결 리스크를 남기지 않는 편이 더 낫다고 봤기 때문이다.

## 9. 시간대 해석

현재 매수 가능 시간은 `09:30 ~ 14:00`이다.

최근 리플레이에서는 `11:30`, `13:00`보다 `14:00`까지 열어둔 쪽이 더 나은 결과를 보였다.  
다만 시간대별 손익을 보면 `11시대`가 상대적으로 약했고, `13시대`가 오히려 강하게 나오는 구간도 있었다.

즉 "오후 진입이 무조건 나쁘다"기보다는, 특정 시간대의 패턴을 더 세밀하게 봐야 하는 상태다.

## 10. 기록과 복기

현재 전략은 단순 자동매매보다 "나중에 분석 가능한 로그"를 꽤 중요하게 본다.

주요 기록:

- `fills`
- `orders`
- `market_traces`
- `account_traces`
- `trade_fills_audit.csv`

특히 `market_traces`에는 아래 정보가 함께 쌓인다.

- 기대수익률
- 스프레드
- 시가총액
- 거래대금
- KOSPI 변화율
- 스캔 시각
- 보유 중 추적용 `active_position`

백테스트 결과 CSV는 `Daily_bot/backtest/results`에 따로 둔다.  
실거래 로그와 백테스트 결과를 분리해 두는 이유는, 실운용 기록과 실험 결과를 섞지 않기 위해서다.

## 11. 최근 복기 포인트

현재 설정 기준 최근 리플레이에서는 다음 특징이 확인됐다.

- 표준 리플레이 총합: `+10.6058%`
- 현재 실거래 로직에 더 가까운 리플레이 총합: `+11.0196%`
- `2026-06-11`은 수익이 크게 났지만, `005090` 한 종목 기여가 매우 컸다.
- `005090`을 제외하면 `2026-06-11` 손익은 `+10.6140%`에서 `+4.1196%`로 내려간다.

따라서 최근 결과를 해석할 때는 단순 총합뿐 아니라,

- 특정 종목 쏠림
- 시간대별 성과
- 손절 반복 종목

까지 같이 보는 편이 맞다.
