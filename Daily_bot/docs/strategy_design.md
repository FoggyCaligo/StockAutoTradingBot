# Daily Bot Strategy Design

## 1. 전략 목적

이 전략은 KOSPI200 후보군 중에서 당일 호가 구조상 단기 기대수익률이 높은 종목을 골라 진입하고, 매수 직후 목표가 매도 주문을 걸어 당일 안에 청산하는 데 목적이 있다.

핵심 가정은 다음과 같다.

- 20호가 스냅샷만으로도 단기 가격 균형점을 추정할 수 있다.
- 기대수익률이 충분하고 스프레드가 과도하지 않은 종목만 고르면, 장이 좋지 않은 날에도 초단기 회전으로 기회를 만들 수 있다.
- 전략 잠재수익률은 기존 보수적 추정보다 높을 수 있다.
- 다만 기록과 손익 계산은 브로커 원본 기준으로 엄격하게 검증해야 한다.

## 2. 후보군 생성

### 기본 우주

- 기본 유니버스: `KOSPI200`
- 시가총액 하한: `2500억 원`
- 거래대금 하한: `30억 원`
- 추세 필터: 현재 비활성화

예전 문서 기준보다 후보군이 더 넓다.  
최근 튜닝에서는 "후보 자체가 너무 적게 잘려 나가 좋은 기회까지 사라지는 문제"를 줄이는 방향을 우선했다.

### 추세 필터

현재는 `trend_filter.enabled = false` 상태다.

즉, 과거에 사용하던:

- 이동평균 기반 추세 확인
- 초기 약세 종목 제거

같은 보조 필터는 현재 주전략 기준에서는 꺼두고 있다.

## 3. 예상가 산출

전략의 핵심은 20호가 기반 예상가 계산이다.

요약 흐름:

1. 각 종목의 20호가 스냅샷을 조회한다.
2. 매수 잔량과 매도 잔량이 균형을 이루는 가격대를 탐색한다.
3. 균형점이 깨지는 frontier를 찾는다.
4. 그 구간의 중간값을 예상가로 계산한다.
5. 틱 사이즈에 맞춰 반올림한다.

목표 매도가:

```text
target_sell_price = expect_price - 1 tick
```

기대수익률:

```text
(target_sell_price - current_price) / current_price * 100
```

## 4. 매수 조건

현재 운영상 중요한 조건은 다음과 같다.

- 기대수익률 `0.30%` 이상
- 스프레드 `0.7%` 이하
- `top_ratio = 0.20` 안에 드는 후보
- 동적 기대수익률 하한 `max(0.3, spread_percent * 1.5)`
- 전영업일 상승률 `7.0%` 이상 종목 제외
- 목표 매도가가 현재가보다 높아야 함
- 계좌 보유 종목이나 미체결 주문과 중복되지 않아야 함
- 주문가능현금으로 1주 이상 매수 가능해야 함

과거 문서에는 `0.25%`, `0.5%`, `0.7%` 같은 단일 문턱 위주 설명이 있었지만, 현재는 스프레드가 큰 종목일수록 더 높은 기대수익률을 요구하는 동적 필터를 함께 쓴다.

또한 현재는 `selected`보다 `unselected` 결과를 기준으로 성과를 해석하는 편이 더 맞다.  
이유는 최근 필터 구조를 많이 손봐서 예전 `selected` 결과가 현재 구조를 제대로 대표하지 못할 수 있기 때문이다.

## 5. 주문 흐름

```text
스캔
-> 후보 순위화
-> top_ratio 적용
-> 최종 필터
-> 매수 주문
-> 매수 체결 확인
-> 목표 지정가 매도 주문
-> 체결 대기
-> 체결 또는 손절/강제청산
```

의도적으로 넣지 않은 것:

- 스윙 보유 전략
- 익일 보유 전략
- 매수 후 재판단 기반 재진입 전략

즉, 매수 후에는 "보유 전략"보다 "청산 실행"에 초점이 있다.

## 6. 리스크 관리

현재 주요 리스크 제어:

- 종목당 예산 배분
- 종목당 최대 예산 제한
- 슬롯 수 상한
- 손절 비율
- 일일 손실 제한
- 강제청산 시간
- 마감 후 브로커 대조

현재 슬롯 운영 기준:

- 기본 슬롯 수는 `slot_budget_unit_krw` 기준으로 늘리되, 세션 총 슬롯 수는 `10개`를 상한으로 둔다.
- 현재 최소 슬롯 수는 `3개`다.
- 자본이 `10슬롯`을 넘는 수준에 도달해도 추가 슬롯을 계속 늘리지 않고 `10슬롯 고정`으로 운용한다.
- 이 구조는 자본을 지나치게 한두 종목에 몰지 않으면서도, 후보 수를 무한히 늘리지 않기 위한 절충이다.

현재 손절 기준:

- 기본 손절선은 `매수가 대비 -3.0%`

이 값은 절대 상수라기보다, 최근 실험과 복기 기준에서 가장 균형이 좋았던 운영상 기준선이다.

## 7. 세션 시간 해석

현재 시간 운영:

- 신규 매수 가능: `09:30 ~ 14:00`
- 강제청산 시작: `15:00`
- 브로커 체결 대조: `15:15`

최근 리플레이에서는 `11:30`, `13:00`보다 `14:00`까지 열어두는 쪽이 더 좋은 결과를 보였다.  
해석상으로는:

- 거래 수 자체가 늘었고
- 오후 기회를 더 담을 수 있었고
- 너무 이른 매수 차단으로 인한 기회 상실이 줄어든 것

으로 보는 편이 자연스럽다.

## 8. 체결기록 설계

### 왜 장중 기록만으로는 부족한가

키움 REST는 이벤트 푸시형 체결통보가 아니라 조회형 흐름이므로:

- 체결 반영이 약간 늦을 수 있고
- 장중에는 매도 체결이 로컬 기록에 늦게 반영될 수 있다.

### 현재 해결 방식

- 장중에는 `ka10076`를 폴링
- 주문번호를 체결목록에서 직접 매칭
- `15:15` 이후 브로커 체결 전체를 다시 읽어 로컬 원장을 보정

이 방식은 실시간 완전일치보다는 "마감 후 정합성 확보"에 초점을 둔다.

## 9. 기록과 리플레이

전략 분석이 나중에도 가능하도록 `market_traces`에 문맥 정보를 함께 남긴다.

현재 중요한 추적값:

- 기대수익률
- 스프레드
- 시가총액
- 거래대금
- KOSPI 변동률
- 스캔 시각
- 보유 추적용 `active_position`

백테스트 CSV는 `Daily_bot/backtest/results`에 저장하고, 실거래 로그는 `Daily_bot/logs`에만 남긴다.

## 10. 성과 해석

성과는 세 층으로 구분해서 보는 것이 안전하다.

1. 장중 손익
   MTS 우선
2. 마감 후 실현손익
   `ka10074` 우선
3. 현재 총손익
   `ka10074 + kt00005`

현재 운영 해석에서는 "총매수금액 기준"보다 "시작 예수금/자산 기준"이 체감 성과를 설명하기에 더 적합하다.
