# Daily Bot Strategy Design

## 1. 전략 목적

이 전략은 KOSPI200 후보군 중에서 당일 호가 구조상 단기 기대수익률이 높은 종목을 골라 진입하고, 매수 직후 목표가 매도 주문을 걸어 당일 안에 청산하는 데 목적이 있습니다.

핵심 가정은 다음과 같습니다.

- 20호가 스냅샷만으로도 단기 가격 균형점을 추정할 수 있다.
- 기대수익률이 충분하고 스프레드가 과도하지 않은 종목만 고르면, 장이 좋지 않은 날에도 초단기 회전으로 수익을 낼 수 있다.
- 전략 잠재수익률은 기존 보수적 추정보다 높을 수 있다.
- 다만 기록과 손익 계산은 브로커 원본 기준으로 엄격하게 검증해야 한다.

## 2. 후보군 생성

### 기본 우주

- 기본 유니버스: `KOSPI200`
- 시가총액 하한: `3000억 원`
- 거래대금 하한: `50억 원`
- 추세 필터 활성화 가능

### 추세 필터

현재 구현은 너무 약한 종목을 초기에 덜어내는 용도입니다.

- 최근 이동평균이 우상향인지 확인
- 데이터가 부족하면 제외

## 3. 예상가 산출

전략의 핵심은 20호가 기반 예상가 계산입니다.

요약 흐름:

1. 각 종목의 20호가 스냅샷을 1회 조회
2. 매수 잔량과 매도 잔량이 균형을 이루는 가격대를 탐색
3. 균형점이 깨지는 frontier를 찾음
4. 그 구간의 중간값을 예상가로 계산
5. 틱 사이즈에 맞춰 반올림

목표 매도가:

```text
target_sell_price = expect_price - 1 tick
```

기대수익률:

```text
(target_sell_price - current_price) / current_price * 100
```

## 4. 매수 조건

현재 운영상 중요한 조건은 다음과 같습니다.

- 기대수익률 `0.25%` 이상
- 스프레드 `0.7%` 이하
- 목표 매도가가 현재가보다 높아야 함
- 계좌 보유 종목이나 미체결 주문과 중복되지 않아야 함
- 주문가능현금으로 1주 이상 매수 가능해야 함

이 중 `0.25%`는 운영상 보수적 기준으로 보고 있으며, 전략 잠재수익률 자체를 뜻하지는 않습니다.

## 5. 주문 흐름

```text
스캔
-> 후보 선별
-> 매수 주문
-> 매수 체결 확인
-> 목표 지정가 매도 주문
-> 체결 대기
-> 체결 또는 손절/강제청산
```

의도적으로 넣지 않은 것:

- 스윙 보유 전략
- 장 종료 후 익일 보유 전략
- 매수 후 재판단 기반 재진입 전략

즉, 매수 후에는 "보유 전략"보다 "청산 실행"에 초점이 있습니다.

## 6. 리스크 관리

현재 주요 리스크 제어:

- 종목당 예산 배분
- 종목당 최대 예산 제한
- 슬롯 수 상한
- 손절 비율
- 일일 손실 제한
- 강제청산 시간
- 마감 후 브로커 대조

현재 슬롯 운영 기준:

- 기본 슬롯 수는 `slot_budget_unit_krw` 기준으로 계단식으로 늘리되, 세션 총 슬롯 수는 `10개`를 상한으로 둡니다.
- 자본이 `10슬롯`을 넘는 수준에 도달하면 추가 슬롯을 늘리지 않고 `10슬롯 고정`으로 운용합니다.
- 이 구간에서는 종목당 `500만 원` 상한을 그대로 유지하지 않고, `전체자본 / 10` 기준으로 분할 예산을 잡아 대자본 구간에서도 자금이 놀지 않게 합니다.

현재 손절 기준 해석:

- 기본 손절선은 `매수가 대비 -4%`입니다.
- 이 수치는 임의 상수가 아니라, 지금까지의 실험과 복기에서 손절 빈도와 손실 확대 방지 사이 균형이 가장 좋았던 운영상 최적선으로 취급합니다.
- 따라서 현 시점 문서와 설정은 모두 `-4%`를 기본 기준으로 맞추고, 이후에도 새 실험 결과가 누적되기 전까지는 기준선 변경보다 일관된 검증을 우선합니다.

중요한 점:

- 기록 누락도 리스크로 봅니다.
- 그래서 이 전략은 매매 로직뿐 아니라 체결 복원과 마감 대조까지 포함해 완성됩니다.

## 7. 체결기록 설계

### 왜 장중 기록만으로는 부족한가

키움 REST는 이벤트 푸시형 체결통보가 아니라 조회형 흐름이므로:

- 체결 반영이 약간 늦을 수 있고
- 장중에는 매도 체결이 로컬 기록에 늦게 반영될 수 있습니다.

### 현재 해결 방식

- 장중에는 `ka10076`를 폴링
- 주문번호를 체결목록에서 직접 매칭
- `15:15` 이후 브로커 체결 전체를 다시 읽어 로컬 원장을 보정

이 방식은 실시간 완전일치보다는 "마감 후 정합성 확보"에 초점을 둡니다.

## 8. 성과 해석

성과는 세 층으로 구분해서 보는 것이 안전합니다.

1. 장중 손익
   MTS 우선
2. 마감 후 실현손익
   `ka10074` 우선
3. 현재 총손익
   `ka10074 + kt00005`

수익률 분모는 목적에 따라 다르게 봅니다.

- 총매수금액 기준
- 시작 예수금 기준
- 시작 추정예탁자산 기준

현재 운영 해석에서는 "총매수금액 기준"보다 "시작 예수금/자산 기준"이 체감 성과를 설명하기에 더 적합합니다.
